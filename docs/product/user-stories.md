# 核心用户故事

> 用 "As a / I want / So that" 格式描述 AreaMatrix 的关键用户故事，每个故事附带验收标准。
>
> 阅读时长：约 8 分钟。

---

## 故事编号约定

`US-<Stage>.<编号>` —— 例如 `US-1.03` 表示 Stage 1 的第 3 个故事。

---

## Stage 1（MVP）

### US-1.01 拖拽导入文件

**作为**一个 macOS 用户
**我希望**把任意文件拖入 AreaMatrix 主窗口
**以便于**让它自动归到合适的分类目录

**验收标准**：
- [ ] 主窗口任意位置（除 modal 外）都接受 drop
- [ ] 拖入后弹出 ImportSheet 显示分类预测
- [ ] 用户可在 Sheet 中调整分类与存储模式
- [ ] 确认后文件落位到正确分类目录
- [ ] 操作在 SQLite 留下 `imported` 日志
- [ ] 主窗口列表立即刷新

### US-1.02 选择存储模式

**作为**一个使用者
**我希望**每次拖入时能选 Move / Copy / Index 三种模式
**以便于**根据文件重要性决定是否保留原文件

**验收标准**：
- [ ] ImportSheet 默认显示设置中保存的默认模式
- [ ] 三种模式的语义有清晰的 UI 提示
- [ ] Move：原位置删除、资料库新增
- [ ] Copy：原位置保留、资料库新增
- [ ] Index：原位置保留、资料库不复制、DB 记录原路径

### US-1.03 自动分类（规则）

**作为**一个使用者
**我希望**应用根据文件名和扩展名自动判断分类
**以便于**省去手动选择的负担

**验收标准**：
- [ ] PDF / DOCX 等文档类自动归 docs
- [ ] 含「发票」/「invoice」关键词的文件优先归 finance（关键词优先于扩展名）
- [ ] PNG / JPG 等图片归 media
- [ ] 代码源文件归 code
- [ ] 无规则匹配时归 inbox（未分类）
- [ ] 分类规则可在 `.areamatrix/classifier.yaml` 中查看与编辑

### US-1.04 树状图导航

**作为**一个使用者
**我希望**在侧边栏看到资料库的完整目录树
**以便于**快速切换分类

**验收标准**：
- [ ] 侧边栏显示所有分类目录及其子目录
- [ ] 每个节点显示文件数（含子节点）
- [ ] 显示中文分类名（如「文档」），底层路径仍是英文 slug
- [ ] 点击节点 → 主区列表筛选该节点下文件
- [ ] 树状图在 10 万节点下不卡顿（虚拟化）
- [ ] 资料库改动后树状图自动刷新

### US-1.05 文件列表与排序

**作为**一个使用者
**我希望**主区以表格形式展示当前节点的文件
**以便于**快速浏览与定位

**验收标准**：
- [ ] 列：名称、分类、大小、导入时间
- [ ] 每列可排序、可拖宽
- [ ] 选中行联动详情面板
- [ ] 支持多选
- [ ] 双击文件用系统默认应用打开

### US-1.06 详情面板查看元数据与改动

**作为**一个使用者
**我希望**选中文件后看到它的元数据和历史改动
**以便于**理解这个文件从哪来、经历了什么

**验收标准**：
- [ ] 三 Tab：元数据 / 改动 / 笔记
- [ ] 元数据：路径、大小、SHA256（带复制按钮）、导入时间、存储模式、原始路径（index 模式）
- [ ] 改动 Tab：按时间倒序显示所有 change_log 条目
- [ ] 笔记 Tab：可读写 Markdown 笔记，保存时同步写文件 + DB

### US-1.07 AreaMatrix 概览自动维护

**作为**一个使用者
**我希望**资料库有一份由 AreaMatrix 自动维护的专属概览
**以便于**在 Finder 或编辑器里也能一眼看到目录概要，同时不破坏已有项目文档

**验收标准**：
- [ ] 每次 import / rename / delete 后受影响范围的概览自动重新生成
- [ ] 概览包含：文件总数、总大小、最近导入时间、文件清单、近 30 天改动
- [ ] 默认生成到 `.areamatrix/generated/`，不写入各子目录的 `README.md`
- [ ] 用户可选择在资料库根目录生成 `AREAMATRIX.md`
- [ ] 应用永不覆盖或改写已有 `README.md`

### US-1.08 改动日志（change_log）

**作为**一个使用者
**我希望**应用记录每一个文件的所有改动
**以便于**追溯历史、回答"这个文件什么时候来的"

**验收标准**：
- [ ] 所有写操作（imported / renamed / moved / edited_note / deleted / external_modified）都写入 change_log
- [ ] 详情面板可显示按文件过滤的时间线
- [ ] DB 损坏时不影响文件系统数据

### US-1.09 外部修改自动同步

**作为**一个使用者
**我希望**在 Finder 或终端里改动资料库后，应用 UI 自动反映
**以便于**不被双向同步问题困扰

**验收标准**：
- [ ] 在 Finder 重命名文件后 UI 1 秒内更新
- [ ] 在 Finder 删除文件后 UI 自动移除
- [ ] 在 Finder 添加文件到分类目录后 UI 自动出现
- [ ] 应用自身的操作不会触发外部同步循环
- [ ] 应用关闭期间发生的外部变化在重启后被检测到

### US-1.10 崩溃恢复

**作为**一个使用者
**我希望**即使应用崩溃也不丢失数据
**以便于**信任这个工具

**验收标准**：
- [ ] 拖入 100 个大文件中途强制 kill 进程，重启后没有半成品文件
- [ ] 重启时自动清理 `.areamatrix/staging/` 残留
- [ ] 重启时提示"上次有 N 个文件未完成导入"
- [ ] DB 用 WAL 模式，断电后能恢复

### US-1.11 重复文件检测

**作为**一个使用者
**我希望**拖入已存在的文件时收到提示
**以便于**避免重复存储

**验收标准**：
- [ ] 基于 SHA256 检测重复
- [ ] 拖入重复文件时弹窗提示："此文件已在 `<path>` 中存在"
- [ ] 用户可选：跳过 / 覆盖 / 仍然导入（保留两份）

### US-1.12 设置面板

**作为**一个使用者
**我希望**有一个设置面板调整应用行为
**以便于**适配我的习惯

**验收标准**：
- [ ] 资料库路径（只读 + 「更改…」按钮）
- [ ] 默认存储模式选择
- [ ] 「在 Finder 中显示资料库」按钮
- [ ] 「打开 classifier.yaml」按钮
- [ ] iCloud 检测开关（默认开）

### US-1.13 首次启动向导

**作为**一个新用户
**我希望**首次启动时被引导完成初始设置
**以便于**不被空白窗口劝退

**验收标准**：
- [ ] 介绍页 → 选资料库路径 → 判断新建/接管 → 完成
- [ ] 默认 `~/AreaMatrix/`，可改
- [ ] 允许选择非空目录作为资料库根
- [ ] 选择非空目录时明确提示：不会移动、重命名、删除或覆盖已有内容
- [ ] 检测到 iCloud Drive 路径时给出警告
- [ ] 空目录可创建标准分类目录（docs/code/design/media/finance/inbox + .areamatrix/）
- [ ] 非空目录只创建 `.areamatrix/` 内部结构，并扫描现有内容建立索引
- [ ] 复制默认 classifier.yaml
- [ ] 初始化 SQLite

### US-1.15 接管已有目录

**作为**一个已有大量文件的使用者
**我希望**可以直接把现有文件夹设为 AreaMatrix 资料库
**以便于**不用先搬迁或重排多年积累的内容

**验收标准**：
- [ ] 用户可以选择任意可写目录作为资料库根，包括非空目录
- [ ] 接管流程只创建 `.areamatrix/` 内部文件，不修改已有用户文件
- [ ] 首次扫描把已有文件写入 SQLite，`storage_mode` 记为 `indexed`
- [ ] 已有 `README.md`、`.git/`、项目文件保持不变
- [ ] 接管完成后，树状图按现有目录结构展示
- [ ] 扫描失败时给出可恢复错误，已写入索引可重跑/补扫

### US-1.14 iCloud 兼容

**作为**一个 iCloud Drive 用户
**我希望**资料库可以放在 iCloud Drive 中
**以便于**多设备同步

**验收标准**：
- [ ] 检测到资料库在 iCloud Drive 时仍允许使用
- [ ] 遇到 `.icloud` 占位符时通过 NSFileCoordinator 触发下载
- [ ] 下载未完成时显示「同步中」状态而非报错
- [ ] 多设备并发修改时基于 mtime + hash 协调（最终一致）
